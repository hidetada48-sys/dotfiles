# Skill改善分析レポート
**作成日：2026-05-11**
**参照元：Anthropic公式 The Complete Guide to Building Skills for Claude（33ページ）**

---

## 前提：PDFが定める重要ルール（比較基準）

| ルール | 内容 |
|--------|------|
| `description` | 「何をするか」＋「いつ使うか」の両方必須、1024文字以内 |
| `allowed-tools` | 使えるツールを明示的に制限できる（セキュリティ向上） |
| `compatibility` | 必要な外部ツール・環境の要件を記録するフィールド |
| SKILL.md肥大化 | 5,000語以内を推奨、超える場合は `references/` に分割 |
| ネガティブトリガー | 「Do NOT use for ...」でオーバートリガーを防ぐ |

## 各Skillの診断

### 1. `x-bookmark-analyze` ⚠️ 中

| 観点 | 評価 | 詳細 |
|------|------|------|
| description | ✅ | 何をするか＋トリガーフレーズあり |
| allowed-tools | ❌ 未設定 | BashのみでよいのにAll-tool状態 |
| compatibility | ❌ 未設定 | python3依存だが記載なし |
| オーバートリガーリスク | ⚠️ | 「今週のブックマーク整理して」は別文脈でも発動しうる |

**改善案：**
```yaml
allowed-tools: "Bash, Read"
compatibility: "python3, ~/test2/bookmarks/ にブックマークファイルが必要"
```
descriptionに `Do NOT use for 一般的な整理・ファイル管理タスク` を追加。

### 2. `x-bookmark-weekly` ⚠️ 中

| 観点 | 評価 | 詳細 |
|------|------|------|
| description | ⚠️ | パス `~/test2/bookmarks/` がdescriptionに入っている（環境依存） |
| allowed-tools | ❌ 未設定 | BashのみでよいのにAll-tool状態 |
| compatibility | ❌ 未設定 | rclone・python3・Discord webhookに依存しているが記載なし |
| Skill間連携 | ⚠️ | analyze側は「weeklyの後に使う」と書いているが、weekly側に逆方向の記述がない |

**改善案：**
```yaml
allowed-tools: "Bash"
compatibility: "python3, rclone, ~/test2/scripts/fetch_bookmarks_v2.py が必要。Discordは任意"
```
descriptionからパスを削除し「~/test2/bookmarks/ に保存」→「ローカルフォルダに保存」に変更。

### 3. `x-bookmark-to-notebooklm` 🚨 高

| 観点 | 評価 | 詳細 |
|------|------|------|
| description | ⚠️ | 「Linux・Windows両対応」と書いているが… |
| Windows実態 | ❌ | `cd ~/test2 && python3` はWindowsネイティブでは動かない（WSL必須） |
| compatibility | ❌ 未設定 | playwright・rclone・python3パッケージ群に依存 |
| allowed-tools | ❌ 未設定 | Bashのみで十分 |

**最大の問題：** descriptionが「Windows両対応」と言っているのに、SKILL.md本文にはLinux/macOS前提のコマンドしかない。Windows（miryo）でSkillが発動したとき失敗する。

**改善案：**
```yaml
allowed-tools: "Bash"
compatibility: "Linux/macOS（WSL経由でWindowsも可）。python3, playwright, rclone 必須"
```
descriptionの「Linux・Windows両対応」を「Linux/macOS（WindowsはWSL経由）」に修正。

### 4. `generate-image` 🚨 高

| 観点 | 評価 | 詳細 |
|------|------|------|
| Windowsパス問題 | ❌ | `~/dotfiles/claude/scripts/generate_image.py` はWindowsで別パスになる |
| GOOGLE_API_KEY依存 | ⚠️ | compatibilityに記載すべき |
| トリガーフレーズ | ⚠️ | 「ポスター作って」「イラスト生成して」「絵を描いて」でトリガーされない |
| allowed-tools | ❌ 未設定 | Bash + Read で十分 |

**改善案：**
```yaml
allowed-tools: "Bash, Read"
compatibility: "Linux専用（Windowsでは動作しない）。GOOGLE_API_KEY 環境変数必須"
```
descriptionに「ポスター」「イラスト」「絵」などのトリガーフレーズを追加。
