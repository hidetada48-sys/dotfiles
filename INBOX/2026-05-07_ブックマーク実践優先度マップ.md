### 2. Skill Creator を入れる（環境改善 + 学びの加速器）

7個のSkillを自作済み。このメタスキルを入れると「仕様を話すだけでSkill.mdが自動生成される」ようになり、Skill設計の質が上がる。

```bash
npx skills@latest add anthropics/skills/skill-creator
```

---

### 3. 「Step 0：文脈を先に渡す」を習慣化する（学び・最重要マインドシフト）

B-5記事の肝。SEOの話だが本質は「AIに一般論ではなく専用回答を出させる唯一の方法」。

> AIの回答が一般論で終わるのは、AIの性能の問題ではなく「任せ方の設計」の問題

CLAUDE.mdには個人ルールはある。しかし「今取り組んでいるプロジェクトのコンテキスト」を先に渡す習慣をつける。

**実践テンプレート：**
```
私の現在のプロジェクト：[何を作っているか]
対象：[誰向けか]
今の課題：[具体的に何が問題か]
制約：[技術スタック/環境]
この情報をベースに以降の全作業を行ってください
```

### 4. alwaysLoad 設定（MCPの常時読み込み）（環境改善）

v2.1.121の新機能。Jina・basic-memoryに設定すれば毎回の手動読み込みが不要になる。

```json
"mcpServers": {
  "jina": {
    "...",
    "alwaysLoad": true
  }
}
```

### 6. エスカレート式モデル使い分けを実践する（C-1の行動化）

RTKでトークン節約はしているが、**使うモデルの選択**も意識する。

> Haiku（計画・壁打ち）→ Sonnet（中程度）→ Opus（最終仕上げ・複雑な推論）

90%のタスクはOpus不要。次の1週間、「まずSonnetで試してみる」癖をつけるだけで体感できる。

### 7. Superpowers Plugin を入れる（環境改善）

using-superpowersスキルは既に入っているが、プラグイン版にはTDD・Systematic Debugging・Plan→コード変換など20以上のSkillが追加で入る。

```
/plugin → Discover → "superpowers" で検索
```

（using-superpowersと重複するSkillは上書きされるだけなので安全）

### 8. Git Guardrails を入れる（環境改善・安全性）

dotfilesを複数環境で管理しているので、push・reset --hard・cleanなどの危険なgitコマンドをブロックするフックとして必須。

```bash
npx skills@latest add mattpocock/skills/git-guardrails-claude-code
```
