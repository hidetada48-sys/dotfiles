## 収益化の前提条件（2026年4月時点）

X広告収益（クリエイター収益配分）を受け取るには3条件を満たす必要がある。

| 条件 | 内容 |
|---|---|
| X Premium加入 | Basic以外（Premium以上）必須 |
| フォロワー数 | 500人以上 |
| **インプレッション数** | **過去3ヶ月で500万回以上** |

**重要：** 500万インプレッションは高いハードル。投稿経験ゼロから1年以内は「頑張れば届く」レベル。  
→ X広告収益だけに依存しない。**Xをnoteへの集客チャネルとして使い、note有料記事・コンサルで収益化するルートを並走させる。**

## 2026年Xアルゴリズム 重要ルール

| ルール | 内容 |
|---|---|
| 最重要シグナル | 滞在時間・リポスト・フォロー |
| リプライの価値 | いいねの150倍のスコア |
| **外部リンク** | **本文に入れるとリーチ30〜50%減。必ずリプライ欄に置く** |
| 長文投稿 | 高評価（動画に次ぐ2番手） |
| テキストのみ短文 | 最低評価 |
| 投稿パターン | 量より質。1投稿あたりの滞在時間とリポスト数が重要 |

## フェーズ1（1〜4ヶ月）：投稿習慣の確立

**目標：フォロワー500人・投稿30本以上**

### 投稿パターン（1note記事あたり）

```
① X記事（長文）：noteの内容をほぼそのまま投稿 ← 滞在時間が稼げる・収益単価高
② ショートポスト：記事の核心を1行で ← 拡散の入口
③ リプライ欄：noteのリンクを置く ← リーチを落とさない
```

### やること

- 週1〜2本のnote記事生成に合わせてX記事・ショートポストも生成する
  （testリポジトリのパイプラインに組み込む）
- ブックマーク収集（test2）の情報から「Claude Codeすげえ」系ショートポストも作る
- 投稿後にリプライ欄にnoteリンクを追記する習慣をつける

### ショートポストの型（コピペ用）

```
型1（衝撃発見型）:
Claude Codeで〇〇してみた。〇〇が〇分で終わった。
は？今まで〇〇でやってたの、何だったんだ…。
これ知らない人、マジでもったいない。

型2（Before→After型）:
[Before] 従来の方法で〇〇すると〇時間かかってた。
[After] Claude Codeに任せたら〇分で終了。
やばくない？しかも〇〇まで付いてきた。

型3（問いかけ型）:
ねえ、Claude Codeの〇〇使ったことある？
〇〇ができるんだけど、〇〇で困ってる人は絶対試してほしい。
〇〇だけでできます。
```

## フェーズ2（4〜8ヶ月）：収益化の入口

**目標：note有料記事リリース・フォロワー1,000〜3,000人**

- フォロワー500人達成 → X広告収益の申請条件が1つ揃う（インプレッションは引き続き積み上げ）
- **note有料記事をリリース**（Claude Code活用の実践まとめ等） ← ここが最初の現実的な収益
- 投稿が30本以上貯まったらAI分身（自動投稿）の導入を検討

### AI分身の準備条件
- 過去投稿30〜100本を `my_posts.txt` に集める
- Claude Codeで文体分析 → system_prompt.txt作成
- Pythonスクリプト + X API v2で自動投稿（初期はdry-run必須）

## 自分の強み

| 強み | 内容 |
|---|---|
| パイプライン稼働済み | 情報収集（test2）→ 記事生成（test）が全自動 |
| 一次体験がリアル | Claude Code実用者としての試行錯誤が蓄積されている |
| 希少な目線 | 「非エンジニアが作った」は刺さりやすい |

## 弱み（直視）

| 弱み | 対策 |
|---|---|
| 過去のX投稿がゼロ | 最初の3ヶ月は地道な積み上げが必要 |
| 動画コンテンツなし | テキスト・長文で勝負（後から動画を足す） |
| 500万インプまでX広告収益ゼロ | note有料記事・コンサルで先行して収益化 |
