Xのブックマークを精読し、3観点から実践推奨レポートを生成する。

## ステップ 1：最新ブックマークファイルを取得

```python
python3 -c "
import os, glob
pattern = os.path.expanduser('~/test2/bookmarks/*.md')
exclude = ['_分析', '_analysis', 'index']
files = [f for f in glob.glob(pattern) if not any(x in os.path.basename(f) for x in exclude)]
files.sort(key=os.path.getmtime, reverse=True)
print('\n'.join(files[:3]) if files else 'NOT_FOUND')
"
```

`NOT_FOUND` が返った場合はユーザーに報告して終了。最新ファイルのパスを記録する。

## ステップ 2：実施済み項目を把握

以下で実施済み（✅）の項目を取得する。ファイルが見つからない場合はスキップして次へ進む。

```python
python3 -c "
import os, glob
pattern = os.path.expanduser('~/.claude/projects/*/memory/project_updates_done.md')
files = glob.glob(pattern)
if files:
    with open(files[0], encoding='utf-8') as f:
        print(f.read())
else:
    print('SKIP')
"
```

## ステップ 3：ブックマークを精読

ステップ1のファイルを全文読む。インデックスファイルがあれば先に読んで全体像を把握してから本文を読む。

読みながら以下を把握する：
- 各記事の主題・要点・実践価値
- 繰り返し登場するキーワードやテーマ
- 具体的なアクション（インストール・設定・習慣化など）を含む記事
- 知識・考え方として吸収すべき記事
- 継続してウォッチすべき発信者・トピック

## ステップ 4：内容でグループ分け

固定カテゴリは使わない。今週のブックマーク内容から自然に浮かび上がるテーマでグループ化する。グループ名は内容を端的に表す言葉にする（例：「Skills設計・活用」「MCPサーバー連携」「トークン節約術」「記憶・エージェント設計」など）。同じ記事が複数グループにまたがる場合は最も本質的なグループに分類する。

## ステップ 5：3観点から推奨を整理

各グループについて、以下の3観点から実践内容を抽出する。全グループで全観点を埋める必要はない。そのグループに該当する観点だけ書く。

**🛠 環境改善・向上**
設定変更、ツール導入、スクリプト追加、ワークフロー改善など、自分の環境を直接よくするアクション。インストールコマンドや設定値があれば具体的に書く。

**📚 学習**
読む・理解する・習得すべき内容。すぐ実装できなくても知識として持つべきもの。「何を学べるか」を1行で示す。

**📡 情報収集**
継続してウォッチすべきトピック、発信者アカウント、媒体、技術動向。「なぜウォッチすべきか」を添える。

## ステップ 6：実施済み項目を照合

ステップ2で把握した実施済み項目と照合する。すでに取り込み済みの内容は推奨本文から除外せず、末尾の「✅ 実施済み」セクションにまとめて記載する。

## ステップ 7：レポートを生成・保存

以下のフォーマットでレポートを作成し、`~/test2/bookmarks/` に保存する。

**ファイル名**：`YYYY-MM-DD_ブックマーク分析.md`（今日の日付）

---

### レポートフォーマット

```
# ブックマーク分析レポート
**作成日：YYYY-MM-DD**
**元データ：[ファイル名]（XX件）**

---

## [グループ名1]

### 🛠 環境改善・向上
- **[アクション名]**：[説明。コマンドや設定値があれば記載]（元記事：X-N）

### 📚 学習
- **[記事タイトル概要]**：[何を学べるか]（元記事：X-N）

### 📡 情報収集
- **[発信者/トピック]**：[なぜウォッチすべきか]

---

## [グループ名2]
...（以下同様）

---

## ✅ 実施済み（参考）
- [すでに取り込んだ内容の一覧]

---

## 今週のピックアップ3選
インパクトが最も高いと判断した3つを、理由とともに一言で紹介する。

1. **[タイトル]** — [理由]
2. **[タイトル]** — [理由]
3. **[タイトル]** — [理由]
```

保存後、ファイルパスをユーザーに伝える。
