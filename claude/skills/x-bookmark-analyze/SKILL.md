Xのブックマークを精読し、3観点から実践推奨レポートを生成する。

## ステップ 1：最新ブックマークファイルとインデックスファイルを取得

```python
python3 -c "
import os, glob, re

pattern = os.path.expanduser('~/test2/bookmarks/*.md')
exclude = ['_分析', '_analysis', 'index']
files = [f for f in glob.glob(pattern) if not any(x in os.path.basename(f) for x in exclude)]
files.sort(key=os.path.getmtime, reverse=True)

if not files:
    print('NOT_FOUND')
else:
    main_file = files[0]
    # インデックスファイルを対応する日付から特定
    m = re.search(r'bookmarks_(\d{4}-\d{4})\.md', os.path.basename(main_file))
    index_file = ''
    if m:
        idx = os.path.join(os.path.dirname(main_file), f'bookmarks_index_{m.group(1)}.md')
        index_file = idx if os.path.exists(idx) else ''
    print(f'MAIN:{main_file}')
    print(f'INDEX:{index_file}' if index_file else 'INDEX:NONE')
"
```

`NOT_FOUND` が返った場合はユーザーに報告して終了。メインファイルとインデックスファイルのパスを記録する。

## ステップ 2：インデックスファイルを先読みして全体像を把握

インデックスファイルが存在する場合（`INDEX:NONE` でない場合）は必ず先に全文を読む。

把握すること：
- カテゴリ別の件数・比率（件数が多いカテゴリ = 今週の主要テーマ）
- よく出るキーワード
- グループ分けの参考にすること（件数比率を無視しない）

インデックスが存在しない場合はスキップして次へ進む。

## ステップ 3：実施済み項目を把握

以下で実施済み（✅）の項目を取得する。ファイルが見つからない場合はスキップして次へ進む。

```python
python3 -c "
import os, glob
pattern = os.path.expanduser('~/.claude/projects/*/memory/project_updates_done.md')
files = glob.glob(pattern)
if files:
    with open(files[0], encoding='utf-8') as f:
        print(f.read())
else:
    print('SKIP')
"
```

## ステップ 4：現在の環境を把握

以下を確認し、ステップ 6 の推奨作成時に「この環境に対して何が有効か」を判断する基準にする。

```bash
# 既存スキル一覧
ls ~/.claude/skills/

# 主要設定ファイルの確認
cat ~/.claude/CLAUDE.md | head -60
```

把握すること：
- 既存スキル（すでに持っているもの vs 持っていないもの）
- CLAUDE.mdの現在のルール・構成
- 推奨を出す際は「この人の環境で今すぐ使えるか」を必ず照合する

## ステップ 5：ブックマークを精読

ステップ 1 で取得したメインファイルを全文読む。ステップ 2 で把握したカテゴリ比率・主要テーマを念頭に置きながら読む。

読みながら以下を把握する：
- 各記事の主題・要点・実践価値
- 繰り返し登場するキーワードやテーマ
- 具体的なアクション（インストール・設定・習慣化など）を含む記事
- 知識・考え方として吸収すべき記事
- 継続してウォッチすべき発信者・トピック

## ステップ 6：内容でグループ分け

固定カテゴリは使わない。今週のブックマーク内容から自然に浮かび上がるテーマでグループ化する。グループ名は内容を端的に表す言葉にする（例：「Skills設計・活用」「MCPサーバー連携」「トークン節約術」「記憶・エージェント設計」など）。

**注意：** ステップ 2 で確認したカテゴリ件数が多いテーマは、必ずグループとして立てる。件数が多いのに分析から落とすのは禁止。

同じ記事が複数グループにまたがる場合は最も本質的なグループに分類する。

## ステップ 7：3観点から推奨を整理

各グループについて、以下の3観点から実践内容を抽出する。全グループで全観点を埋める必要はない。そのグループに該当する観点だけ書く。

**ステップ 4 で把握した現在の環境を照合し、「この人の環境で具体的に何をするか」まで落として書くこと。**

**🛠 環境改善・向上**
設定変更、ツール導入、スクリプト追加、ワークフロー改善など、自分の環境を直接よくするアクション。インストールコマンドや設定値があれば具体的に書く。既存のSkillsやCLAUDE.mdへの適用方法を明示する。

**📚 学習**
読む・理解する・習得すべき内容。すぐ実装できなくても知識として持つべきもの。「何を学べるか」を1行で示す。

**📡 情報収集**
継続してウォッチすべきトピック、発信者アカウント、媒体、技術動向。「なぜウォッチすべきか」を添える。

## ステップ 8：実施済み項目を照合

ステップ 3 で把握した実施済み項目と照合する。すでに取り込み済みの内容は推奨本文から除外せず、末尾の「✅ 実施済み」セクションにまとめて記載する。

## ステップ 9：優先順位をつける

全推奨アイテムに以下の基準で優先度タグを付ける。

- **[P1]** 今すぐ・1コマンドまたは1設定変更で試せるもの
- **[P2]** 近いうちに着手すべきもの（既存環境の改善・新しい使い方の導入）
- **[P3]** 中長期でウォッチ・習得すべきもの（トレンド・情報収集）

今週のピックアップ3選はP1を優先して選ぶ。P1がない場合はP2から選ぶ。

## ステップ 10：レポートを生成・保存

以下のフォーマットでレポートを作成し、`~/test2/bookmarks/` に保存する。

**ファイル名**：`YYYY-MM-DD_ブックマーク分析.md`（今日の日付）

---

### レポートフォーマット

```
# ブックマーク分析レポート
**作成日：YYYY-MM-DD**
**元データ：[ファイル名]（XX件）**

---

## [グループ名1]

### 🛠 環境改善・向上
- [P1] **[アクション名]**：[説明。コマンドや設定値があれば記載。既存環境への適用方法も明記]（元記事：X-N）

### 📚 学習
- [P2] **[記事タイトル概要]**：[何を学べるか]（元記事：X-N）

### 📡 情報収集
- [P3] **[発信者/トピック]**：[なぜウォッチすべきか]

---

## [グループ名2]
...（以下同様）

---

## ✅ 実施済み（参考）
- [すでに取り込んだ内容の一覧]

---

## 今週のピックアップ3選
P1優先で、インパクトが最も高いと判断した3つを理由とともに一言で紹介する。

1. **[タイトル]** — [理由]
2. **[タイトル]** — [理由]
3. **[タイトル]** — [理由]
```

保存後、ファイルパスをユーザーに伝える。
